Safety is a product feature

Powerful changes deserve visible boundaries.

WimForge is designed around an explicit review contract. The original source stays protected by default, commands remain structured, and consequential work crosses confirmation gates only after the plan is inspectable.

01Clone by default

Writes target project-owned working paths.

02Hash the inputs

Changed or missing files stop the graph.

03Journal recovery

Interrupted mounts return for safe review.

04Publish atomically

Partial output never masquerades as done.

Honest scope: WimForge orchestrates Windows and provider tools; it cannot remove the need for backups, correct licensing, Windows SIM validation, or disposable VM testing.

From a clean Windows machine

One command to bootstrap, build, test, and package.

The maintained bootstrap detects a clean existing checkout or plans a canonical clone, repairs user-profile tools under the normal identity, requests UAC only when machine package repair is needed, then returns to install Qt archives, clone, build, test, package, hash, and log the release without elevation.

01 Detect or clone 02 Install supported tools 03 Build and test 04 Package and log

Fail closed: dirty checkouts, hidden Git index state, unsafe build paths, incomplete tools, failed tests, and missing artifacts stop the run. A unique clone pinned to the verified commit isolates each build from ignored files and stale packages. Existing source is never reset or cleaned, every log is retained, and packaged files finish with SHA-256 evidence.

Normal PowerShell · UAC if repair is needed
$ErrorActionPreference='Stop'; $d=Join-Path ([IO.Path]::GetTempPath()) ('WimForge-bootstrap-'+[Guid]::NewGuid().ToString('N')); $s=Join-Path $d 'bootstrap-build.ps1'; $ps=Join-Path ([Environment]::SystemDirectory) 'WindowsPowerShell\v1.0\powershell.exe'; New-Item -ItemType Directory -Path $d -ErrorAction Stop | Out-Null; try { Invoke-WebRequest 'https://raw.githubusercontent.com/Ding-Ding-Projects/WimForge/main/scripts/bootstrap-build.ps1' -UseBasicParsing -OutFile $s; & $ps -NoLogo -NoProfile -ExecutionPolicy Bypass -File $s; if ($LASTEXITCODE -ne 0) { throw "WimForge bootstrap failed with exit code $LASTEXITCODE." } } finally { Remove-Item -LiteralPath $d -Recurse -Force -ErrorAction SilentlyContinue }
⌾

Review before elevation. Run this from a normal, non-administrator Windows PowerShell session. Inspect bootstrap-build.ps1 first, or download it and run -Plan. This convenience command follows mutable main; replace main with a reviewed commit SHA when reproducibility matters.

Required outside the script: Windows 10/11 x64, 64-bit PowerShell 5.1, UAC consent or administrator credentials, Microsoft App Installer/WinGet for the invoking user, internet access to GitHub, WinGet and Qt archives, and adequate disk space. Proxy, policy, licensing, vendor, reboot, or network failures are reported with a retained log rather than bypassed. Exact package IDs still resolve catalog versions current at run time, so logs and hashes are traceability evidence—not a promise of identical toolchains across dates.
